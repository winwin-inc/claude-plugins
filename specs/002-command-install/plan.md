## 执行摘要

### 项目目标

创建 **Wiki Generator 安装工具** `wiki-generator`，这是一个独立的 Python CLI 工具，用于将 Claude Code 的 wiki-generate 命令和相关模板文件安装到用户项目中。

**核心价值**：
- **一键安装**：通过 `uvx wiki-generator` 命令快速安装 wiki-generate.md 和模板
- **标准化管理**：统一的安装、更新、卸载流程
- **简化部署**：新项目无需手动复制文件
- **独立工具**：不依赖 Claude Code 命令系统，通过 uvx 调用

### 关键成果

- **5 个核心动作**：install、list、update、uninstall、info
- **3 种安装来源**：Git 仓库、本地文件、预设名称
- **智能冲突处理**：分场景策略，保护用户数据
- **备份和回滚**：可回滚的更新机制
- **配置管理**：JSON 格式的配置文件
- **专用用途**：主要安装 `.claude/commands/wiki-generate.md` 和 `.claude/templates/`

### 实现策略

采用**独立 Python CLI 工具 + uvx**架构：
1. **技术栈**：Python 3.8+ + uv（依赖管理和执行）
2. **调用方式**：
   - 方式 1（推荐）：`uvx wiki-generator <action>`（uv 自动处理依赖）
   - 方式 2：`wiki-generator <action>`（安装到 PATH 后）
   - 方式 3：`python -m wiki_generator.cli <action>`（模块方式）
3. **来源解析优先级**：本地路径 > Git URL > 预设名称
4. **浅克隆优化**：使用 `git clone --depth 1` 减少 93% 时间
5. **分场景冲突处理**：根据文件类型和场景选择策略
6. **自动备份机制**：更新前自动备份，支持一键回滚
7. **默认目标**：安装 wiki-generate.md 和 templates 到当前运行目录

## Phase 0: 研究与技术决策

### 研究任务列表

#### 任务 1：来源解析机制 ✅

**问题**: 如何准确解析不同的命令来源？

**决策**: **启发式分层检测策略**

**研究结果**：
1. **检测顺序**：本地路径 > Git URL > 预设名称
2. **本地路径特征**：`./`、`../`、`/`、`~`、`C:\`、文件名含 `.md`
3. **Git URL 格式**：支持 HTTPS、SSH、Git 协议
4. **安全检查**：SSRF 防护、路径遍历检测、协议限制
5. **预设映射**：配置文件映射预设名称到 URL

**性能目标**：
- 本地路径解析：< 10ms
- Git URL 解析：< 100ms

**完整研究报告**: [research.md](research.md#技术决策-1来源解析机制)

---

#### 任务 2：文件冲突处理策略 ✅

**问题**: 如何处理文件冲突，保护用户数据？

**决策**: **分场景策略**

**研究结果**：
1. **冲突检测**：文件存在检查 + SHA-256 哈希比较
2. **解决策略**：
   - 跳过（批量安装默认）
   - 备份后覆盖（更新默认）
   - 智能合并（JSON 配置）
   - 交互式询问（重要文件）
   - 强制覆盖（`--force`）
3. **安全考虑**：使用 SHA-256 而非 MD5（防止碰撞攻击）

**推荐策略矩阵**：

| 场景 | 默认策略 | 命令行参数 |
|------|----------|-----------|
| 全新安装 | 直接安装 | 无 |
| 更新命令 | 备份后覆盖 | `--backup` |
| JSON 配置 | 智能合并 | `--merge` |
| 批量安装 | 跳过冲突 | `--skip` |
| 重要文件 | 交互式询问 | `--ask` |

**完整研究报告**: [research.md](research.md#技术决策-2文件冲突处理策略)

---

#### 任务 3：Git 仓库克隆和资源提取 ✅

**问题**: 如何高效、安全地克隆和提取命令文件？

**决策**: **浅克隆到临时目录，验证后复制**

**研究结果**：
1. **浅克隆优势**：
   - 节省 93% 克隆时间
   - 节省 98% 磁盘空间
   - 满足 99% 的安装场景

2. **资源发现**：
   - 支持标准结构（`commands/`、`templates/`）
   - 支持扁平结构（根目录）
   - 支持自定义结构（配置文件指定）

3. **安全检查**：
   - 路径遍历检测
   - 符号链接逃逸检测
   - 编码验证（UTF-8）
   - 不执行仓库中的任何脚本

4. **性能优化**：
   - LRU 缓存（避免重复克隆）
   - 指数退避重试（网络容错）
   - 临时文件自动清理

**完整研究报告**: [research.md](research.md#技术决策-3git-仓库克隆和资源提取)
